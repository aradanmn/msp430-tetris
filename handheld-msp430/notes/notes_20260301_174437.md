### Session 4 (2026-03-01) — THIS SESSION

#### Root Cause Diagnosed: Off-Grid Component Positions
Screenshot analysis revealed small red marks at ALL IC pin endpoints — these are KiCad's "dangling wire" indicators. Wire stubs are present but don't electrically connect to pins.

**Root cause**: Component positions (175, 125, 60, 280, …) are NOT multiples of 2.54 mm.

```
Example: U1 was at cx=175 mm
  175 / 2.54 = 68.897…  ← NOT on grid
  Pin endpoint: 175 - 7.62 = 167.38 mm
  167.38 / 2.54 = 65.90…  ← NOT on grid
  Wire start: 167.38 mm   ≠ pin location in KiCad's grid
  → KiCad sees wire as disconnected from pin
```

**Fix**: All positions expressed as integer multiples of G = 2.54 mm.

```
U1 now at cx = 69×G = 175.26 mm
  175.26 / 2.54 = 69.0  ✓
  Pin endpoint: 175.26 - 7.62 = 167.64 mm
  167.64 / 2.54 = 66.0  ✓
  Wire/label all snap to grid  ✓
```

#### Changes in gen_kicad7.py
1. `G = 2.54` constant defined at top
2. ALL component positions expressed as `n*G` (integer multiples)
3. `body_hw = 2*G` for all ICs (including LM386, previously 3.81 = 1.5G)
4. `STUB = G` (wire stub = 1 grid unit)
5. POT pin offset changed to `3*G = 7.62 mm` (previously 3.81 = non-grid)
6. Grid validation loop: prints warning if any position off G grid
7. Pin endpoint spot-check at end: confirms all sample pins on 1.27 mm grid
8. Button rows: BTN_YS = [36G, 40G, 44G, 48G, 52G, 56G, 60G, 64G] — centred near U2

### Required KiCad 9 fields on placed symbols
```scheme
(symbol (lib_id "NAME") (at cx cy 0)
  (unit 1)
  (exclude_from_sim no)   ← required in KiCad 9
  (in_bom yes)
  (on_board yes)
  (dnp no)                ← required in KiCad 9
  ...)
```

### Label format
```scheme
(label "NET_NAME" (at x y rot)
  (fields_autoplaced yes)
  (effects (font (size 1.27 1.27)) (justify left))
  (uuid "..."))
```
- Connection point IS at (x, y) regardless of rot
- rot=0: flag points left, text extends right (for right-side IC pins)
- rot=180: flag points right, text extends left (for left-side IC pins)
- rot=270: flag points down, text above (for top pins)
- rot=90: flag points up, text below (for bottom pins)

### Hide syntax (KiCad 9, format 20241004+)
```scheme
(effects (font (size 1.27 1.27)) (hide))   ← bare boolean
;; NOT: (effects ... (hide yes))           ← KiCad 8 syntax, INVALID in 9
```

## Grid Alignment Rules (CRITICAL — learned from session 4 debugging)

**All component positions MUST be multiples of G = 2.54 mm.**

Why: KiCad uses 50 mil (1.27 mm) fine grid. Pin endpoints = cx ± (body_hw + G).
For pin endpoints to land on 1.27 mm grid:
- cx must be multiple of G = 2.54 mm (since body_hw is also multiple of 1.27 mm)
- cy must be multiple of G = 2.54 mm (since pin ly offsets are multiples of 1.27 mm)

If ANY position is off-grid, KiCad's wire stubs don't touch pin endpoints → ERC errors → "nothing connected."

**Verification**: Run the grid check loop in gen_kicad7.py before writing the file.

### Coordinate System
```
Symbol local: Y increases UP (math coords)
Schematic world: Y increases DOWN (screen coords)
Formula: world = (cx + lx,  cy - ly)   [Y is NEGATED]

Pin endpoint offset from IC centre:
  lx = ±(body_hw + G) = ±(2G + G) = ±3G = ±7.62 mm  (for body_hw=2G)
  ly = (n-1)×1.27 - i×2.54  (pin i, 0-indexed, n total pins per side)
```
